<CATS>

title - заголовок задачи;
lang* - язык описания задачи;
tlimit - лимит времени на прохождения одного теста (в секундах);
mlimit – лимит памяти на прохождение одного теста (в мегабайтах)

author* - автор;
inputFile - имя входного файла;
outputFile - имя выходного файла;
difficulty* - авторская оценка сложности задачи в баллах от (1 до 100);
stdChecker* - имя стандартной программы проверки;

Стандартная программа проверки служит для сравнения выходного файла с файлом, содержащим ответ теста.
Перечень стандартных программ проверки:

  • nums - проверяет совпадение целых чисел (4 байта - long int);
  • floats2 - проверяет совпадение вещественных чисел (10 байт - long double);
  • strs - проверяет совпадение строк до 256 символов;
  • longnums - проверяет совпадение длинных целых чисел (до 10000 цифр);
  • longstrs - проверяет совпадение длинных строк (до 10000 символов);

name - идентификатор объекта <Checker> (произвольная текстовая строка)
src* - путь к файлу с исходным кодом;
de_code* - код среды разработки.

Описание проверяющей программы. Данный тэг игнорируется, если в описании задачи присутствует атрибут stdChecker.
"src" - задает путь к файлу, относительно корня архива. Допускается использование вложенных каталогов. Регистр пути к файлу должен совпадать с регистром имен файлов в архиве.
"de_code" - задает среду разработки.
Перечень стандартных кодов сред разработки:

  • 101 - Cross-platform C/C++ compiler (cpp;c)
  • 102 - GNU C++ (cpp;c;cxx)
  • 103 - MS Visual C++ 7.0 (cpp;c)
  • 104 - Borland C++ 3.1 (cpp;c) 
  • 201 - Borland Pascal (pas) 
  • 202 - Free Pascal 1.0.10 (pp;pas)
  • 203 - Borland Delphi 7.0 (pas)
  • 301 - Quick Basic 4.5 (bas)  (больше не поддерживается)

Если код среды разработки не указан, то он определяется по расширению файла, в порядке приоритета, заданного списком стандартных кодов сред разработки.

Описание генератора больших тестов.

<Test>

<Problem>

rank - порядковый номер теста.

Тэг содержит описание тестов. Rank - должно пробегать все значения от 1 до некоторого N. Если N = 0, то любое компилируемое решение будет принято как верное.

<TestRange>

<Problem>

from – порядковый номер первого теста

to – порядковый номер последнего теста

Внутри тега TestRange допускается использование контекстных переменных: %n и %0n.

%n – номер текущего теста

%0n – номер текущего теста с лидирующим нулем

Например:

<TestRange from=”1” to=“10”><In src=”%0n.in”/><Out use=”solution”/></TestRange>
